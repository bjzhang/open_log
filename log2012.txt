shell脚本和正则表达式入门
上次用perl处理hexdump结果时已经用到了正则表达式。今天用Android pad shell 直接处理hexdump结果，这样省去了在pc上执行脚本和复制到pad的流程。
为了简单起见，咱们只是分析第一个脚本，上次这个脚本是这样的：
#!/usr/bin/perl 
while(<>) {
    chomp;
    #0000000 043c 0000 830e 0007 0003 0030 0001 0000
    #serial second     usecond   type code value
    if (/^[0-9a-f]+\ ([0-9a-f]+)\ ([0-9a-f]+)\ ([0-9a-f]+)\ ([0-9a-f]+)\ ([0-9a-f]+)\ ([0-9a-f]+)\ ([0-9a-f]+)\ ([0-9a-f]+)/ ) {
        print "sendevent /dev/input/event3 $type  $code $value\n";
}

这里面核心就是if这句话。

1, 正则表达式是个什么东东？
正则表达式简单说就是增强的通配符。一般用在搜索和替换中。
通配符这个东西平时咱们也用过。比如
ls kernel*
是列出所有以kernel开头的文件。
还有印象吧，原来为了取保存的日志咱们曾经这样看过有哪些kernel的log.
adb shell ls /system/kernel*
后来把所有日志都放在一个目录里面，就不需要看文件名再adb pull了。
PS: adb shell下ls每次都有乱码，这些乱码其实是显示颜色的控制字符。可以用"ls --color=never"不显示颜色，这样就没有乱码了。
alias ls="ls --color=never", 所有ls命令都会变成"ls --color=never". 
\todo 举例。

2, 最简单的正则表达式
ls log20[0-9][0-9].txt
列出log20开头的后面是两位数字的后缀是txt的文本文件。
例如log2009.txt, log2010.txt, log2011.txt
如果是通配符是个什么结果呢？
ls log20*.txt
基本会列出所有文件。
再举例，只想保存特定的log.

常用的正则表达式元字符：
. + * [] () {}
.
任意一个字符，不包括换行。
+
一个或多个，所以".+"表示一个或多个任意字符。"0+"表示一个或多个0.
*
零个或多个，所以".+"表示零个或多个任意字符。"0+"表示零个或多个0.
[]
匹配其中任意一个字符。例如[01]表示匹配0或1. [0-9]匹配0123456789中任意一个数字。所以[0-9a-z]+匹配一个或多个十六进制数。呵呵这个比较有用啊。

1, ref: http://superuser.com/questions/391052/paste-format-string-or-paste-with-replace

it's not completely clear to me what you want, but in Vim you could do something like this:

:for i in range(line('$')) | call append(i+i, printf("this is line %d:", i+1)) | endfor
The reason I use "i+i" is that, as lines are inserted into the buffer the line numbers change, and this corrects for that. (If you have a different "starting point" in your range, your correction algorithm will have to be more complicated.)

Or if you want to modify the existing lines in the buffer instead:

:for i in range(1, line('$')) | call setline(i, printf("this is line %2d: %s", i, getline(i))) | endfor
Of course, if you don't want to change the buffer, but just print to the screen, use "echo" instead of "call" and remove the append() / setline() chunks from around the printf().

See:

:help :for
:help :call
:help :echo
:help range()
:help line()
:help append()
:help setline()
:help getline()
:help printf()

A2: Here's the trick: First, set your vi(m) session to allow pattern matching with special characters (ie: newline). It's probably worth putting this line in your .vimrc or .exrc file.

:s/,/,^M/g

离职136定理发布时间：2012-04-19 13:09:15  
http://forum.eet-cn.com/BLOG_ARTICLE_12046.HTM
最近公司离职率很高，除了主动离职还有被动离职，行业的洗牌越来越明显，大牌的公司通过福利的提升吸引了一大批本来实干的工程师们。老板一边抱怨这是由于队伍的不成熟导致的，希望找来一批30岁以上的工程师，他们不像出来1~5年的工作经验的人，老是吃碗里的看兜里的。可是HR反馈30岁以上的工程师实在很难找。最近看到一个离职分析的微博，136定理，一个月内离职跟HR有密切关系，3个月离职跟上司有密切关系，6个月离职跟公司文化有密切关系，1年后离职跟自己的晋升空间有关系，3年后离职跟发展平台有关。6年后基本就不会离职了。
另一方面反观拜倒在糖衣炮弹下的工程师的共同特点：1.有家室开支更大（有选择的欲望） 2.年龄在28左右（工作5年内）3.熟悉现在的公司（或许比领导人更明白自己的位置和公司的位置）4.技术上有一定得本钱经得起跳槽的考验（有选择的本钱） 
看来28左右是一个很关键的门槛，一方面有了家庭压力，特别是在房价高起，生活各处均有RMB做主的年代（需要通过跳槽进一步开发自己的价值）。另一方面社会的浮躁，从08年的4万亿计划开始，人们的生活开始发生了变化，大量的印钞让国民消费力大大削弱，地产作为支柱产业收到各路投机客的加入。投机市场越来越热各类艺术品的天价拍卖。当一个社会的人均以投机为主，忽视实业的时候，社会形成投机文化，像工程师这种技术实干的职业将会受到社会越来越多的诱惑。 
这种职业的倦态，你的身上有吗？

关于安卓系统使用蓝牙键盘的问题
http://bbs.hiapk.com/thread-591952-1-1.html
Android手机Nexus One连接Apple Bluetooth Keyboard
Posted on August 20, 2010 by henry
首先所有带蓝牙的android手机理论上都是可以连接蓝牙键盘鼠标的(Human Interface Device)，不幸地是官方的蓝牙profile中默认不包括HID profile。所以只有采用第三方的ROM如Cyanogen（CM6版本以上）或在官方的rom上搭配bluez utility来实现蓝牙键鼠的连接。
这里我们仅讨论第二种在官方ROM上添加工具的方案。手机必须取得root权限，或者是ADP (Android Dev Phone)，并且需要安装busybox。
1. 先下载需要的工具。
$ wget http://androidobex.googlecode.com/files/hciconfig
$ wget http://androidobex.googlecode.com/files/hcitool
$ wget http://androidobex.googlecode.com/files/hidd
2. 提升到root权限，并且将/system挂载成可读写，默认为read-only
$ su
# mount -o remount,rw -t yaffs2 /dev/block/mtdblock3 /system
# chmod 777 /system/xbin
3. 从本地复制第一步下载的文件到手机
$ adb push hidd /system/xbin
$ adb push hcitool /system/xbin
$ adb push hciconfig /system/xbin
4. 如果你的手机不是ADP的话，可能无法直接从本地电脑复制文件到手机。这样你可以先用usb线连接手机，把以上三个文件复制到sdcard根目录上，然后执行。
# cp /sdcard/hidd /system/xbin/
# cp /sdcard/hcitool /system/xbin/
# cp /sdcard/hciconfig /system/xbin/
5. 打开手机蓝牙，在Settings -> Wireless & network settings -> Bluetooth settings中找到键盘并且配对。pin code可以填写0000，配对成功后会发现该设备显示为’paried but not connected’，这是因为缺少HID profile。
6. 现在要用到之前拷入手机的三个文件。实际情况中设备的Mac地址会与下面的例子有所不同，根据实际情况输入。
# hciconfig
hci0: Type: UART
BD Address: 00:22:A5:B8:AD:65 ACL MTU: 1021:4 SCO MTU: 180:4
UP RUNNING PSCAN
RX bytes:8672 acl:98 sco:0 events:285 errors:0
TX bytes:3336 acl:102 sco:0 commands:89 errors:0
# hcitool dev
Devices:
hci0 00:22:A5:B8:AD:65
# hcitool scan
Scanning …
00:1D:4F:A7:9A:49 Apple Wireless Keyboard
# hidd –connect 00:1D:4F:A7:9A:49
# hcitool con
Connections:
< ACL 00:1D:4F:A7:9A:49 handle 1 state 1 lm MASTER
9. 至此，Apple Bluetooth Keyboard连接成功。打开任意程序，蓝牙键盘敲击的键将出现在输入栏。经过测试，谷歌中文法也可以正常使用并且可以用数字键选择候选字。
10. 剩下的问题就是苹果的键盘的多功能键（调节音量，屏幕亮度）暂时还不能在手机里使用，有待进一步研究如何修改这几个键位。

2, ИСУ-152
http://bbs.ainol.com/forum.php?mod=viewthread&tid=15895&extra=&page=3
楼主，有几个疑问，还望解答或者帮忙打听下：
首先芯片设计一般包括前端设计、后端设计、设计验证、流片、硬件功能测试等几个环节。
前端设计根据难度不同一般耗时都是数月之久，即使全志的CPU核心内核和GPU均是购买的ARM公司现成的IP，但是VPU等其他外设却是全志自行设计。即使是沿用A10上面的结构而不做丝毫改进那么前端设计至少也会耗时一个月。当一个芯片的结构定了之后，即使工艺发生变化，一般来说也可以沿用同一套前端设计，但是后端设计和设计验证却需要和采用的工艺对应，也就是说换一次工艺就需要重新进行一次后端设计和相应的设计验证，这两样工作的总耗时也至少需要一个月的时间。而在完成设计验证之后就可以进行流片，流片周期一般为两个月左右，在芯片完成流片以后还需要进行硬件功能测试以便验证芯片是否达到设计预期，这一工作耗时大约是半个月，同时也是转换一次流片工艺即需要进行一次硬件功能验证。那么如此算来，若真是按照楼主所说经历40nm、32nm、28nm三次不同工艺的流片，那么最少也需要1个月的前端设计，3*1个月的后端设计及设计验证（总共3个月），3*2个月的流片时间（总共6个月），3*0.5个月的硬件功能验证（总共1.5个月），这样算下来就是说全志四核至少在11个半月前就开始了设计。况且近年来全志一直在台积电进行流片工作，而众所周知台积电并没有32nm工艺生产线，要进行32nm流片必然需要在其他代工厂进行。撇开产能不谈，光是转换代工厂并熟悉代工厂的工艺特点带来的后端设计周期及设计验证周期增长就是以月计算的。如果真如楼主所说，那么全志四核至少应该在一年多以前就开始设计了，而一年前的此时此刻，全志的A10处理器才刚刚上市，还不用说之后还经历了A13的设计工作，而全志公司的规模显然不具备使用多个IC设计组并行设计多个完全不同结构CPU的能力。况且多次流片的费用也非常可观，尤其是随着工艺的进步，晶圆成本也大幅上升，据统计12寸的28nm晶圆的成本是12寸65nm晶圆的1.4倍。而全志的制胜法宝就是低成本，因此使用28nm显然不是最佳选择；32nm由于台积电没有该工艺，而新寻找一家没有合作过的代工厂显然风险过大，且产能也是一个较大的问题。因此我觉得最大可能行还是使用台积电的40nm工艺进行制造...以上疑问还请楼主进一步确认是否属实。

三防参数含义
据了解，同为三防，360特供机“超级战舰”的三防级别为ip54，而MOTO DEFY的三防级别为ip67。别看指数上细微的差别，但对于手机这种电子产品来说，可谓天壤之别。“超级战舰”的ip54代表防尘，并不防止全部尘土进入，但进入量不能达到妨碍设备正常运转的程度。对于防水，也仅限任何方向的溅水。而MOTO DEFY的三防级别为ip67，代表可防尘土进入，并可浸沉在规定压力的水中经规定时间后进入水量无害。
简单的说，同样在宣传时出现的“三防”二字，其背后的意义完全不同。MOTO DEFY即使将手机丢进面盆、浴缸都可以“全身而退”，而“超级战舰”则最多能保证在下雨时不会损害手机。“像我这种下雨导致手机进水的情况可能由于放在衣兜里，长时间浸水造成的。因为‘超级战舰’只防溅水，而不能保证浸水后手机‘健康’。”白先生无奈的说。

White Anglo-Saxon Protestant (WASP) is an informal term, often derogatory or disparaging, for a closed group of high-status Americans mostly of English Protestant ancestry. The term implies the group as wielding disproportionate social and financial power.[1] The term WASP does not describe every Protestant of English background, but rather a small restricted group whose family wealth and elite connections allow them a degree of privilege held by few others. When the term appears in writing, it usually indicates the author's disapproval of the group's perceived excessive power in society. People seldom call themselves WASPs, except humorously; the acronym is typically used by non-WASPs.
Scholars agree that the group's influence has waned since the end of World War II, with the growing importance of Jews, Catholics, and other former outsiders.[2] The term is also used in Australia and Canada for similar elites.[3][4]

Hi Jian,

This use of ARM support requires an active support contract. I'll provide a quick answer:

The virtualization extensions are not configurable and are part of the A7. There are certain aspects of virtualization and LPAE that are design specific. As an example it is not required for an SOC designer to implement and use the external 40bit physical addresses. Also IF an external interrupt controller is used then connection of 'virtual' interrupts must be supported by the SOC designer.

1, http://baike.baidu.com/view/590.htm
人体预存的ATP能量只能维持极限强度运动大约2秒，随后由CP合成ATP，大约能维持6秒，合计8秒左右。也就是说，全速跑不到一百米即告罄，跑二百米时后面的一百米，必须由血糖在无氧状态下，迅速合成新的热能物质ATP来提供能量，其副产品是乳酸。跑二百米或四百米、一百米游泳、网球和足球等运动，是利用肌糖原无氧分解所提供的能量，故运动后肌肉里累积大量乳酸，乳酸堆积是运动后引起肌肉痛的原因之一。这类运动所需的糖由糖原提供，故也烧不到脂肪，这不是有氧运动,对减肥无益。
　　肌糖原无氧分解所提供的能量，只能维持一分钟左右，跑完四百米后就全部用完。跑八百米时，后面的四百米，必须由糖、脂肪酸和氨基酸在有氧状态下，合成新的热能物质ATP来提供能量，而糖由糖原分解后供应，脂肪酸由脂肪分解后供应，氨基酸由蛋白质分解后供应，这整个过程需要氧气，也就是靠氧气燃烧糖、脂肪和蛋白质来生产热能物质ATP，供应后段运动所需的热量，这后段的运动就是有氧运动。跑八百米或一千五百米、二百和四百米游泳、拳击等运动，都需要开始利用氧气燃烧糖原、脂肪和蛋白质，故此类运动的后段都是有氧运动，作为有氧运动，心率一般在130次/分为最佳。运动的前段大约五分钟先烧糖原，运动持续越久会烧掉越多的脂肪，只要持续半小时至一小时，所消耗热量的五成，就由燃烧脂肪来供应。

SDXC的標誌

SDXC（Secure Digital eXtended Capacity）為SD卡的新一代標準，於2009年CES消費電子展提出。

SD 3.0規格於低容量中之SDXC卡仍使用FAT32，SD 3.0規定之最高速度為832 Mbit/s（UHS104[1]），而高容量則於SD 4.0中規範最大容量可高達2TB，採用的檔案系統是微軟專利的exFAT規格。SD 4.0規範預定將此速度提升至2.4 Gbit/s。SDHC所採用的FAT、FAT32、Ext2於SDXC仍可使用，SDXC電子介面規格仍與SDHC相容。
目录
UHS，全稱Ultra High Speed，是與SDXC同時推出的SD卡讀寫速度標準。此標準適用於SDHC和SDXC。
目前只有UHS-I一個標準，最高傳輸速度（理論值）為104MB/s。英文字母I代表該裝置（SD卡或讀卡器）支援UHS-I介面。英文字母U，包含數目字1，代表該裝置讀寫速度達UHS-I。
注意，裝置必須同時印有UHS 1.png和UHS I.png，讀寫速度才有UHS-I。

美国加州的科学家在最新一期《自然》杂志上发表文章，揭开中药常山的更多秘密。
两千年多来，中医一直使用俗称常山的根茎治疗疟疾。最近的研究发现，常山的活性成分常山酮可用于治疗多种自体免疫疾病。研究人员此前发现，常山酮触发了一种应激反应路径，阻止了一类有害的免疫细胞Th17发育，这类细胞被认为与许多自体免疫疾病有关联。最新报告进一步显示，常山酮通过阻断转移核糖核酸分子（transfer RNA）的生成，消灭人体免疫系统内的“坏”细胞，保护“好”细胞，消灭人体血液内的疟疾寄生虫，从而治愈疟疾以及由其引起的发热。常山酮现已被小范围运用在治疗癌症和肌肉萎缩症的人体试验中。
